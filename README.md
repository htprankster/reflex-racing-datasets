Official reflex.racing datasets generated by the [utils repo](https://github.com/htprankster/reflex-racing-utils).
<hr>

## Datasets
### Leaderboards
There are 2 types of datasets:
1. <ins>**custom**</ins>
   - Contains extra information in a JSON format about maps and players that the ```racestats.sqlite``` database does not include.
   - All properties, except **_tags_** (players) and **_custom_** (maps) which are manually declared, are regularly updated by a script using the Steam web api.
2. <ins>**latest**</ins>
   - JSON datasets are created by exporting the leaderboards database (.sqlite) and merging them using the <ins>**custom**</ins> objects compiled on the previous step.
   - This archive is managed by the script and as such, manual edits will get overwritten.
### RSS
This dataset merges all RSS feeds retrieved using the rules declared in ```streamList.json``` from the code repo.<br>
No changes are made to the content.

## Data moderation
Players caught cheating will have all of their records removed from the JSON dataset, following a reindex of all rankings.

Game developers are aware of the exploits and are working on the proposed fixes, in the meantime, the racing community admins will handle moderating the leaderboards.

Banned players are not allowed to appeal their ban as long as the current game version doesn't include a remediation for that specific cheat, following either with a verified manual overwrite of their records or a leaderboards reset.

Unmoderated leaderboards are still accessible without the extra features on the provided ```racestats.sqlite``` database or its extracted json files with prefix ```ldb_noedit_*.json```.

## Transparency
You can verify the authenticity of the leaderboards by replacing the datasets into the utils repository and running the script with another database link:

*Before starting, make sure to match both the utils repository and the datasets versions at the time of the generation you want to verify*
1. Download this archive into a ```/external``` folder on the root of the utils repository.
2. Rename the ```racestats.sqlite``` database.
3. Replace its path to the ```ldb``` variable inside ```/scripts/steamworks/generateLeaderboards.sh``` script.
4. Run the script from the container.
5. Compare the generated datasets sha256 checksums with those committed here on ```metadata.json```
